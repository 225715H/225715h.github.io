実際にやってみる

参考にしたサイト

上記を参考にgithubでのサイト生成を行なった。

雛形の作成

まず、hugoコマンドでサイトのディレクトリを作る。

(base) ~ $ hugo new site gitsite2
中略

Just a few more steps...

1. Change the current directory to /Users/.../...
2. Create or install a theme:
   - Create a new theme with the command "hugo new theme <THEMENAME>"
   - Install a theme from https://themes.gohugo.io/
3. Edit hugo.toml, setting the "theme" property to the theme name.
4. Create new content with the command "hugo new content <SECTIONNAME>/<FILENAME>.<FORMAT>".
5. Start the embedded web server with the command "hugo server --buildDrafts".

See documentation at https://gohugo.io/.

作ったディレクトリに移動する。

(base) ~ $ cd gitsite2
(base) ~ $ ls
archetypes/ content/    hugo.toml   layouts/    themes/
assets/     data/       i18n/       static/

hugoテンプレートのクローン

サイトにしたいhugoテンプレートをthemesの中にクローンする。

今回はMainroadを使用する。 (サイトのデモ)

(base) ~ $ git clone https://github.com/Vimux/Mainroad.git themes/mainroad

テンプレートの中にあるexampleSiteのファイルの中身をコピーする。

(base) ~ $ cp -r -f Mainroad/exampleSite/* ../
(base) ~ $ cd ..
(base) ~ $ ls
archetypes/  config.toml  data/        i18n/        static/
assets/      content/     hugo.toml    layouts/     themes/

hugo.tomlはいらないので削除して大丈夫。

一度、hugo new siteで作ったディレクトリ内で、hugo server でサイトが反映されているか確認する。

静的サイトのディレクトリの生成

同じディレクトリでhugoコマンドを打つことで、githubにpushすべきディレクトリ(今回はpublic)が作成される。

githubでwebページ公開用のリポジトリを作成する。

リポジトリ名は<ユーザー名>.github.ioとすること。

hugo new siteで作ったディレクトリの親ディレクトリにgithubで作成した空リポジトリをクローンする。

(base) ~ $ cp -r public/* ../<ユーザー名>.github.io
(base) ~ $ cd ../<ユーザー名>.github.io
(base) ~ $ git add .
(base) ~ $ git commit -m "first commit"
(base) ~ $ git push origin main

これで、https://<ユーザー名>.github.io/にアクセスすると見れるようになっている。
