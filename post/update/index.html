サイトの更新

サイトを生成した後の記事更新について記載していく。

新しいポストを作る

hugo new <content配下のディレクトリ>/タイトル.mdコマンドでポストファイルを作成できる。

hugo serverで変更を確認しながら、編集を進める。

できたら、hugo new site で作ったディレクトリ(筆者はgitsite2)でhugoを実行し、publicの中身を更新する。

サイトを更新する

その後、publicをgithubリポジトリのディレクトリにコピーする。

(base) ~ $ cp -r public/* ../<ユーザー名>.github.io
(base) ~ $ cd ../<ユーザー名>.github.io
(base) ~ $ git add .
(base) ~ $ git commit -m "update"
(base) ~ $ git push origin main

これで、反映されるが、多少時間がかかる。

後日談

サイトの更新をgitsite2内にシェルスクリプトでまとめた。

(base) ~ $ cat hugopush.sh 
#!/bin/zsh
cd ~/os/gitsite2
hugo
cp -r public/* ../225715h.github.io
cd ../225715h.github.io
git add .
git commit -m "post update"
git push
sleep 5
echo "completed push"
