概要

プロジェクトの作成

任意のフォルダでdjango-admin startproject <ディレクトリ名>を実行することでプロジェクトを作成することができる。

作成されたディレクトリに移動し、python3 manage.py runserverを実行し、以下の画面が出れば成功である。

ファイルの説明

(base)~ $ ls
HelloWorld/ db.sqlite3  manage.py*
(base)~ $ cd HelloWorld
(base)~ $ ls
__init__.py  __pycache__/ asgi.py      settings.py  urls.py      views.py     wsgi.py

作成したプロジェクトの中身は上記のようになっている。

基本は以下の3つのファイルを編集する。

  • HelloWorld/HelloWorld/setting.py: このファイルは、クラス定義した場合にパスを設定したり、アプリ作成した場合に、アプリのディレクトリのパスを設定するファイルである。

  • HelloWorld/HelloWorld/urls.py: このファイルは、サーバからのリクエストを受け、レスポンスする関数やクラスを記述するファイルである。

  • HelloWorld/HelloWorld/views.py: このファイルは、レスポンスするために反映する関数やクラスを記述するファイルである。

function based view

function based viewとは、大まかにいうと表示を細かく指定して実装するもの。

つまり、views.pyで関数を定義してその関数自身を呼び出すことができる。

class based view

class based viewとは、大まかにいうと表示をテンプレートに頼ったような簡単に実装できるもの。

つまり、views.pyでクラスを定義することで他で作成したディレクトリのファイルを呼び出すなどができる。

アプリの作成

python3 manage.py startapp <ディレクトリ名>でアプリを作成することができる。

今は、表示するファイルが増えたときの役割をはっきりするための階層化みたいな解釈で大丈夫(?)

実際にやってみる

目標

上記のような表示を目標とする。

関数を使う

urls.pyを以下のように編集する。

from django.contrib import admin
from django.urls import path
from .views import helloworldfunction

urlpatterns = [
    path('admin/', admin.site.urls),
    path('HelloWorld/', helloworldfunction)
]

pathの引数1には任意のURLconfを指定する。引数2には反映させる関数を設定する。関数はviews.pyに記述していくのでfrom .views import <関数名>となっている。

その後、views.pyを以下のように記述することで表示される。

from django.http import HttpResponse
from pathlib import Path

def helloworldfunction(request):
    example = HttpResponse('<h1>Hello World!</h1>')
    return example

クラスを使う

クラスを使う場合、htmlに記述されたhello worldを表示する。

まず、urls.pyを以下のように編集する。

from django.contrib import admin
from django.urls import path
from .views import HelloWorldClass

urlpatterns = [
    path('admin/', admin.site.urls),
    # path('HelloWorld/', helloworldfunction),
    path('class/', HelloWorldClass.as_view()),
]

呼び出すクラスの後ろに.as_view()をつける。(おまじないみたいなもの)

クラスをインポートするのを忘れないこと。

その後、setting.pyのDIRS:[]にBASE_DIR / 'templates'と記述する。

BASE_DIRはプロジェクトを作ったディレクトリを変数としてる。その中に,templatesというフォルダを読み込むというパスを設定している。

TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [BASE_DIR / 'templates'],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        },
    },
]

以下になるようにtemplatesフォルダを作成する。

作成したフォルダに、反映させたいhtmlファイルを作成する。 今回は簡単に以下のように記述する。

(base) ~ $ cat HelloWorld/templates/hello.html 
hello world

views.pyは以下のように記述する。

from django.http import HttpResponse
from django.views.generic import TemplateView
from pathlib import Path

# def helloworldfunction(request):
#     example = HttpResponse('<h1>Hello World!</h1>')
#     return example

class HelloWorldClass(TemplateView):
    template_name = 'hello.html'

汎用ビューであるTemplateViewをインポートすることで、template_nameで指定したファイルのテンプレートを表示することができる。

変更後、python3 manage.py runserverを実行し、任意のURLconfを入力して目標の出力が得られたらOK
