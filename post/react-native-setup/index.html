環境設定

nodeコマンドの設定

nvs(Node Version Switcher)を使う。

export NVS_HOME="$HOME/.nvs"
git clone https://github.com/jasongin/nvs "$NVS_HOME"
. "$NVS_HOME/nvs.sh" install

よく使うnvsコマンド

  • nvs ls-remote: nodeの全てのバージョンは表示する

  • nvs add lts(or latest): 最新のlts(Long Term Supprot)または、最新バージョンをダウンロードする。

  • nvs ls: 現在使用できるバージョンが表示される。

  • nvs use lts: addしたバージョンを指定することでnodeコマンドが使用できる。

settingからiOSをインストールしておく。

Androidのインストール

Android Studioをダウンロードする。

基本的なものは、初めのセットアップでダウンロードできるはず。

以下は確認である。More ActionsからSDK Managerを選択する。

上部にあるSDK Platformsを選び、APIが一番最新のものをダウンロードする。

次は、SDK Toolsを選び、画像のように選択されてるものを使えるようにダウンロードしておく。

Simulatorは、More ActionsからVirtual Device Managerを選択する。

上記の+ボタンから新しくsimulatorを作成する。

アプリを作りたいの端末を選択する。基本はPixelの最新で大丈夫。

基本は、Recommendedにあるのを選択すればいいが、ない場合はOther Imagesから選択する。

残りは起動するsimulatorの名前をつけたら、Finishで作成完了。

最後に ~/.zshrcファイルに以下を追加する。

# Android Studio
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$ANDROID_HOME/platform-tools:$PATH
export PATH=$ANDROID_HOME/emulator:$PATH

adb --versionでバージョンが確認できれば設定完了。

watchmanのインストール

brewコマンドを使って、watchmanをインストールしていく。

brew install watchman

その後、brew listでインストールできているか確認する。

動作確認

アプリを作成したいディレクトリに移動し、nvs use ltsでnodeコマンドを使えるようにしてプロジェクトを作成する。

npx create-expo-app --template
✔ Choose a template: › Blank (TypeScript)
✔ What is your app named? … <ディレクトリ名>
✔ Downloaded and extracted project files.

初めの項目は、Blank (TypeScript)を選択する。次の項目は、プロジェクト名をつけてあげればディレクトリとして作成される。

その後、作成されたディレクトリに移動し、npx expo startでsimulatorを起動できる。

npx expo startでサーバが立てられない場合、npm installを実行するとできるようになる。
